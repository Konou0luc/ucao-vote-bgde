Le projet VoteBGDE n’est pas encore terminé.

Nous allons maintenant développer un deuxième backend séparé : l’API universitaire officielle de l’UCAO-UUT.

IMPORTANT :
Cette API est indépendante du système de vote.

Architecture globale du projet :

1. API Universitaire
Responsable de :
- étudiants
- inscriptions
- filières
- niveaux
- statut académique
- vérification des matricules

2. API VoteBGDE
Responsable de :
- OTP
- votes
- participation
- scrutins
- résultats
- administration

Le système VoteBGDE devra consommer l’API universitaire via HTTP/API REST.

OBJECTIF DU NOUVEAU PROJET

Développer une API universitaire professionnelle avec :
- Node.js
- TypeScript
- Express
- Prisma ORM
- PostgreSQL
- Swagger
- Clean Architecture
